# 作业操作手册 —— 从本地到 GitHub 的完整流程

> 本手册按顺序执行即可完成作业全部评分要素。
> 预计总耗时：30–40 分钟。

---

## 前置准备

确认本机已安装：

```bash
git --version
mvn -v          # Maven 3.6+
java -version   # JDK 11
```

没有 Git 的话，到 https://git-scm.com/download/win 下载安装；
没有 Maven / JDK 的话，上次单元测试实验已经装过，应当无需重复。

---

## 第 1 步：在本地验证项目

在 `C:\Users\44479\Documents\软件测试\测试管理` 下运行：

```bash
mvn -B test
```

**预期**：会看到 `multiplyMixed` 等 3 个用例失败（这是注入缺陷，正常现象）。这证明测试能跑，也证明 bug 可复现。

> 如果你希望把 bug 先"修好"让 CI 绿一次再改回红色演示，见本手册末尾的「补充：演示修复前后对比」。

---

```bash

git config --global user.name  "hdz0318"

```

---

## 第 3 步：在 GitHub 上创建仓库

1. 浏览器打开 https://github.com/new
2. **Repository name**：`complex-unit-test`
3. **Description**：`软件测试课程 - 测试管理作业：单元测试 + 持续集成 + 缺陷管理`
4. **Public**（公开，方便助教查看）
5. **不要** 勾选 *Add a README* / *.gitignore* / *license*（本地已有）
6. 点击 **Create repository**

---

## 第 4 步：推送到 GitHub

复制 GitHub 给出的远程地址（应为 `https://github.com/hdz0318/complex-unit-test.git`），回到终端：

```bash
git remote add origin https://github.com/hdz0318/complex-unit-test.git
git push -u origin main
```

如果弹出身份验证：推荐用 GitHub 网页 → 右上头像 → *Settings* → *Developer settings* → *Personal access tokens (classic)* 生成一个勾选 `repo` 权限的 token，在 push 时把它作为密码输入。

---

## 第 5 步：查看 CI 运行 & 冒烟测试日志

1. 进入仓库首页 → 点击顶部的 **Actions** 标签
2. 你会看到一次 `CI - Build & Smoke Test` 运行
3. 点开 → 展开 **Run smoke tests (mvn test)** 步骤
4. 能看到类似：
   ```
   [INFO] Running org.apache.commons.math3.complex.ComplexArithmeticTest
   [ERROR] Tests run: N, Failures: 3, Errors: 0, Skipped: 0
   ```
5. **截图保存** → 这是评分点「持续集成中的冒烟测试」的证据。
6. 底部 Artifacts 区域下载 `surefire-reports` → 里面的 XML 可作为 bug 证据附件。

---

## 第 6 步：登记缺陷 (Issue)

1. 仓库页面 → **Issues** 标签 → **New issue**
2. 选择 `Bug report / 缺陷报告` 模板
3. 把 `BUG_REPORT.md` 的正文**整体粘贴**到 Issue 里（按模板里的章节填入）
4. 右侧：
   - **Labels**：`bug`（如需要再加一个 `critical`）
   - **Assignees**：自己
   - **Milestone**（可选）：新建一个 `v1.0.1` 演示里程碑管理
5. 点 **Submit new issue**
6. **截图保存** Issue 页面。

---

## 第 7 步：演示"从 Issue 到修复"的闭环（加分项）

在本地修复 `Complex.java:252`：

```java
// before
real * factor.imaginary - imaginary * factor.real
// after
real * factor.imaginary + imaginary * factor.real
```

提交并推送：

```bash
git add src/main/java/org/apache/commons/math3/complex/Complex.java
git commit -m "fix: correct sign in Complex.multiply imaginary part (closes #1)"
git push
```

- commit message 里的 `closes #1` 会让 GitHub 在 CI 通过后**自动关闭对应 Issue**
- 再看一次 Actions，这次应该是 ✅ 全绿
- 截图：修复后的 Actions 绿色运行 + 自动关闭的 Issue

## 补充：演示修复前后对比（可选）

如果老师要求"持续集成通过测试发现问题 + 修复后通过"，按以下顺序：

1. 第一次 push：保留缺陷 → CI 红 → 截图 → 开 Issue
2. 第二次 push：`fix: ...` commit → CI 绿 → 截图 → Issue 自动关闭

这样一条流水线上能看到 `bug → Issue → fix → close` 完整闭环，非常加分。
